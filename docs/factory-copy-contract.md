# T3-420 Web Factory Copy Contract

This document defines the safe copy contract for using T3-420 Web as the master template for Factory-generated sites.

Factory must copy only the public web runtime needed by the generated instance. It must not copy local development state, private configuration, logs, database temp files, dumps, or secrets.

## Files Factory Should Copy

- `index.html`
- `css/`
- `js/`
- `assets/`
- `manifest.webmanifest`, either generated by Factory or copied from the base template
- `service-worker.js`, either generated by Factory or copied from the base template
- `worker.js`, only when the generated instance is deployed with the Cloudflare Worker runtime
- `package.json`, only when it is needed for local preview tooling

## Files Factory Must Not Copy

- `.env`
- `.env.*`
- `.git/`
- `.wrangler/`
- `node_modules/`
- `supabase/.temp/`
- `logs/`
- `backups/`
- private docs
- test files
- `css/style copy.css`
- temporary files
- dumps, including `*.dump`
- SQL exports not explicitly required by the target instance
- secrets or files containing secrets

## Values Factory Must Replace

Factory must replace or generate these values for each generated club instance:

- Supabase URL
- Supabase anon key
- Telegram username
- `manifest.webmanifest`
- service worker cache name
- service worker cache prefix
- logo
- home/background image
- PWA icons
- `wrangler.jsonc` `name`

## Supabase Values

The browser may receive only public Supabase values:

- Supabase project URL
- Supabase anon key

Factory must never place these values in frontend files:

- `service_role_key`
- database passwords
- private API keys
- provider secrets
- Telegram bot token
- Twilio auth token
- WhatsApp access token

## Service Worker And Cache

Factory must generate a cache name that is unique per club slug and version.

Recommended shape:

```text
t3-420-<club-slug>-pwa-v<version>
```

Factory must also generate a cache prefix unique to the club slug so one generated instance cannot delete another instance cache when both run under the same origin.

## Deployment Notes

`worker.js` contains asset blocking and runtime environment injection for Cloudflare deployments. If an instance is hosted without this Worker, Factory or the host must still prevent private files from being published.

The current demo may contain T3-420 demo configuration. Generated instances must not inherit demo identifiers unless the target instance is intentionally the T3-420 demo.
